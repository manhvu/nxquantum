## Required Step Outcomes (for F2 Scenarios)

For each currently scaffold-only step module:

1. Parse and handle all Given/When/Then clauses in the corresponding feature file.
2. Assert typed envelope fields and deterministic metadata shape.
3. Assert representative error cases and deterministic codes.
4. Fail fast on unhandled steps (no passive capture behavior).

## Unit/Property Linkage Guidance

For each F2 feature family, add/expand supporting tests under:

1. `test/nx_quantum/` for contract and behavior unit tests.
2. `test/property/` for deterministic invariants where applicable.

Minimum linkage targets:

1. Dynamic capability preflight determinism.
2. Calibration payload normalization/typed diagnostics.
3. Topology policy deterministic reporting fields.
4. Simulation fallback determinism and tolerance envelope behavior.
5. Batch ordering/shape determinism and scalar-parity checks.

## Milestone Execution Checkpoints

### Milestone L (P0)

1. Existing provider bridge step modules remain green after adapter seam hardening.
2. No public API contract changes in step assertions.

### Milestone M (P1)

1. All five scoped step modules are executable and assertion-driven.
2. `mix test.features` validates all mapped feature files with no scaffold-only step modules.

### Milestone N (P2)

1. Mapping remains aligned with `docs/v0.6-acceptance-criteria.md`.
2. Docs and release evidence references are consistent and build cleanly.

## Non-Goals (v0.6 Mapping)

1. Introducing centralized orchestration behavior into NxQuantum.
2. Adding dashboard/UI/control-plane concerns to feature scenarios.
3. Expanding scope into unscheduled v0.8 migration assurance APIs.
