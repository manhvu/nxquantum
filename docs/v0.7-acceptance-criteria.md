## Feature Acceptance Criteria

### F1) External Operations Integration Contract v1

Required outcomes:

1. Capability/lifecycle/error envelopes remain versioned and machine-readable.
2. Correlation and idempotency fields are explicit and deterministic.
3. Contract serialization is stable across equivalent inputs.
4. Public contract changes follow explicit versioning policy.

Acceptance checklist:

1. Typed envelope shapes are schema-tested for version `v1`.
2. Equivalent request inputs produce identical serialized envelope structures.
3. Missing or malformed required envelope fields fail with typed deterministic diagnostics.
4. Release notes include migration note whenever envelope schema changes.

### F2) Observability Export and Schema Hardening

Required outcomes:

1. Observability schema stability is enforced across `high_level`, `granular`, and `forensics`.
2. Redaction and cardinality safeguards remain contract-level requirements.
3. Export-capable and no-op/test adapter strategies preserve functional behavior contracts.

Acceptance checklist:

1. Mandatory span/metric/log keys are acceptance-covered for all profiles.
2. Cardinality guardrails block unbounded label growth in default profile paths.
3. Sensitive fields remain deterministically redacted in logs/diagnostics.
4. Observability adapter substitution does not change lifecycle result contracts.

### F3) Standalone Production Hardening + Integration Profiles

Required outcomes:

1. Standalone production profile guidance is executable and contract-aligned.
2. External integration profile guidance is executable and contract-aligned.
3. Integration examples demonstrate typed capability preflight and error handling.
4. No external orchestration dependency is introduced into NxQuantum runtime usage.

Acceptance checklist:

1. Standalone profile examples execute with current public APIs.
2. Integration profile examples emit machine-consumable contracts and telemetry shapes.
3. Feature and docs references are linked to deterministic scenario evidence.
4. Boundary non-goals remain explicit in v0.7 docs.

## v0.7 Milestone Gates (Acceptance-Driven)
